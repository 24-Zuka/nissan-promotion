# Google カレンダー連携の設定

CRM の「期限のあるタスク（点検・車検・営業フォロー）」を、**あなたの Google カレンダーへ
一方向で書き出す**機能です。サーバーを介さず、ブラウザから直接あなたのカレンダーに書き込みます
（クライアント秘密鍵は不要・無料）。

> 仕組み: Google Identity Services（GIS）のトークン方式。アクセス権はブラウザ内に一時保持され、
> 顧客データはあなたの端末とあなたのカレンダーの間だけで完結します（サーバーには渡りません）。

---

## 1. Google Cloud で OAuth クライアント ID を作る（初回だけ・1回）

1. https://console.cloud.google.com/ にログイン → 上部でプロジェクトを新規作成（任意の名前）。
2. **「API とサービス」→「ライブラリ」** で **Google Calendar API** を検索 → **有効にする**。
3. **「API とサービス」→「OAuth 同意画面」**:
   - User Type は **外部** を選択 → 作成。
   - アプリ名・サポートメール等を入力（自分用なので最小限でOK）。
   - **「テストユーザー」に自分の Google アカウントを追加**（これでGoogleの審査なしに本人利用可）。
4. **「API とサービス」→「認証情報」→「認証情報を作成」→「OAuth クライアント ID」**:
   - アプリケーションの種類 = **ウェブアプリケーション**。
   - **承認済みの JavaScript 生成元** に、アプリを公開している URL の**オリジン**を追加:
     - GitHub Pages の場合: `https://24-zuka.github.io`
     - ローカル確認の場合: `http://localhost:4100`（など実際に使うポート）
   - 「作成」すると **クライアント ID**（`xxxx.apps.googleusercontent.com`）が発行される。
     これをコピー（**クライアント シークレットは使いません**）。

> 注意: 「承認済み JavaScript 生成元」には**パスを含めない**（`https://24-zuka.github.io/nissan-promotion`
> ではなく `https://24-zuka.github.io`）。

---

## 2. アプリ側で連携を有効化（毎回ではなく初回設定）

1. アプリの **設定** を開く → **「カレンダー連携」をON**。
2. 現れた「Google カレンダー設定」に、手順1でコピーした **クライアント ID を貼り付け** → **保存**。
3. **「Google と接続」** をタップ → Google の同意画面で許可（テストユーザーとして本人なら通る）。
4. **「今すぐ同期」** をタップ → 期限のあるタスクがカレンダーに終日予定として作成される。

以降、タスクを追加・編集・完了・削除したら、設定画面の **「今すぐ同期」** を押すと
カレンダーへ反映されます（完了・削除したタスクの予定は自動で消えます）。

---

## よくある質問

- **何が同期される？** due_date のある未完タスク（点検/車検/営業フォロー）。完了・削除したものは
  カレンダーからも削除されます。予定のタイトル＝タスク名、説明＝顧客名＋詳細。
- **双方向？** いいえ。CRM →カレンダーの一方向です（カレンダー側の変更は CRM に戻りません）。
- **複数端末で使う？** クライアント ID は同じものを各端末の設定に貼ればOK（公開情報です）。
  対応表は端末ごとに持つため、同期は端末ごとに「今すぐ同期」を実行してください。
- **接続に失敗する**: 「承認済み JavaScript 生成元」がアプリの URL オリジンと一致しているか、
  Calendar API が有効か、テストユーザーに自分が入っているかを確認してください。
