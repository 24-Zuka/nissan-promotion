# code-review

## いつ使うか
ユーザーが「レビューして」「この差分を見て」「PRをチェック」と言ったとき。

## 手順
1. 対象差分を特定する（`git diff main...HEAD` 等）。範囲が曖昧なら確認する。
2. 次の優先順で読む: 正確性 → セキュリティ → 挙動の退行 → テスト不足。
3. 指摘は必ず `path:行` を添え、深刻度 HIGH/MEDIUM/LOW を付ける。
4. 良い点を最低1つ挙げる。
5. 結論を3行以内で要約し、マージ可否の所感を述べる（決定は人間）。

## 出力フォーマット
```
## レビュー結果（深刻度順）
- [HIGH] path/to/file.ts:42 — 説明 / 修正案
- [MEDIUM] ...
## 良い点
- ...
## 所感
- （2〜3行）
```
