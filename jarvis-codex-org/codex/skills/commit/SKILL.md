# commit

## いつ使うか
ユーザーが「コミットして」「変更を確定」と言ったとき。

## 手順
1. `git status` と `git diff` で変更内容を把握する。
2. コード変更ならテストを実行し、緑を確認する（赤なら止めて報告）。
3. 変更を論理単位に分け、必要ならステージを分割する。
4. Conventional Commits 形式でメッセージを書く:
   `feat: ...` / `fix: ...` / `docs: ...` / `refactor: ...` / `test: ...` / `chore: ...`
5. 本文に「何を・なぜ」を1〜3行。`main`へのpush/マージは行わず人間に委ねる。

## 注意
- 秘密情報（トークン/鍵）をコミットに含めない。`.env` 等を確認する。
