# daily-brief

## いつ使うか
朝会スクリプトから、またはユーザーが「今日のブリーフィング」と言ったとき。

## 手順
1. 入力を集める: カレンダー予定、未読の要対応、GitHubのCI状況、天気。
   （無料/ローカル認証で取れるものに限る。取れない情報は「未取得」と明記）
2. 各ドメイン（仕事/開発/私用）を並列に確認し、要点を抽出する。
3. 次の構成で日本語のDailyノートを生成する:
   - 今日の最優先 3件
   - 予定タイムライン
   - 開発: 進行中タスク / CI状況 / レビュー待ち
   - 要対応（人間の承認・確認が要るもの）
   - メモ / インボックスからの繰越
4. Obsidian MCP で `Daily/YYYY-MM-DD.md` に書き込む（追記、上書きしない）。

## 注意
- 事実は簡潔に。誇張や脚色をしない（深刻な話題ほど淡々と）。
- 人間の確認が要る項目は「☐ 要承認」を頭に付ける。
